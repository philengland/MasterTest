Observations:

  • Don't terminate a line with the delimiter, as the next <p> element's beginning will already act as a partition
  • What looks like a space character in the XML Editor can sometimes actually be one or many tabs compressed into a single space character visually, depending on source formatting... note the differences in content spacing in the Text Editor between the different sections of the process, despite very little appearance change in XML Editor.
  • The '>' character seems to break the parser and cause the Exception report. See the Comparison section for proof. Can make a Table from the first 3 lines without issue, then see the Exception thrown when including the fourth line. Swap that character and the data becomes a Table as expected.
